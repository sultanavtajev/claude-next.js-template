Dette er bootstrap-kommandoen for templaten. Brukeren kjører denne én gang i et nytt prosjekt, rett etter kloning.

## Instruksjoner

### 0. Verifiser systemforutsetninger

Før du starter sjekklisten, verifisér at nødvendige verktøy er installert. Hvis noe mangler, installér eller be bruker om å gjøre det.

#### Node.js
```bash
node --version
```
Forventet: `v20.x` eller nyere. Hvis versjonen er eldre: stopp og be bruker oppgradere Node.js (https://nodejs.org).

#### pnpm (påkrevd — templaten bruker pnpm)
```bash
pnpm --version
```

Hvis kommandoen feiler med "not found" eller "not recognized":
```bash
npm install -g pnpm
```
Kjør så `pnpm --version` igjen for å bekrefte.

#### git
```bash
git --version
```
Skal bare fungere — git brukes overalt. Hvis ikke: be bruker installere (https://git-scm.com).

#### Valgfrie CLIer (sjekkes senere i respektive steg)
- `gh` (GitHub CLI) — sjekkes i steg 11
- `vercel` (Vercel CLI) — sjekkes i steg 11 hvis Vercel brukes
- `supabase` (Supabase CLI) — installeres som dev-dep i steg 07 Del 4

Rapportér resultatet av sjekk kort til bruker (f.eks. "Node v22.0.0, pnpm v9.0.0, git v2.43 — klart for bootstrap") før du fortsetter til steg 1.

### 1. Les sjekklisten

Les `$CLAUDE_PROJECT_DIR/oppstart/CHECKLIST.md` i sin helhet. Den lister stegene du skal utføre (01–12) og reglene du må følge underveis.

Viktig:
- **Steg 05 (i18n)** kommer før Supabase fordi next-intl sin proxy må settes opp før Supabase-session-refresh chainer på.
- **Steg 06 (Element Picker)** er en liten dev-tool som krever at layout.tsx er flyttet til `[locale]/` først (fra steg 05).
- **Steg 07 (Supabase)** er ett samlet steg som dekker database-klienter, Supabase Auth og RLS — de er tett koblet og behandles sammen.
- **Steg 10 (Prosjekt-struktur)** oppretter `teknisk/`-mappa som `/2.0-task` og `/2.1-task-team` senere skriver sjekklister til.

### 2. Følg reglene i sjekklisten strengt

- Kjør stegene i rekkefølge (01 → 09). Ikke hopp over.
- For hvert steg: les stegfilen først, kjør pre-flight docs-sjekk hvis spesifisert, så eksekver.
- Oppdater `CHECKLIST.md` med `[x]` etter hvert ferdig steg.
- **Alle spørsmål til brukeren skal stilles via `AskUserQuestion`-verktøyet.** Ikke spør i fritekst. Gruppér relaterte spørsmål i samme AskUserQuestion-kall.
- Hvis et steg feiler eller er uklart: stopp, rapporter, og bruk `AskUserQuestion` for å avklare.

### 3. Når alle 12 stegene er grønne

Du er ferdig. Bootstrap er fullført, `oppstart/`-mappen er slettet (i steg 12), første commit er laget. Ikke gjør noe ekstra utover det sjekklisten ber om.

## Hvorfor denne kommandoen eksisterer

Uten denne måtte brukeren skrive en lang instruksjon som "les oppstart/CHECKLIST.md og utfør alle stegene…". Nå holder det med `/0.0-oppstart`.
